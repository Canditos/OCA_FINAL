# Findings
- The project is an existing TypeScript-based OCPP Certification Pipeline with a Dashboard.
- We are adopting the B.L.A.S.T. and A.N.T. architecture moving forward.
- Jira Xray GraphQL and REST API requires internal field IDs ('68f2fbd3a6fdbe3e4e952f0b') instead of field names for custom properties in the tests array.
- Xray evidence arrays accept base64-encoded zip files directly within the test object payload, avoiding generic attachments to the root execution issue.
